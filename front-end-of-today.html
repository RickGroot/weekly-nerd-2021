Front-end developers

De term "Front-End developer" wordt steeds vaker gebruikt. Je komt de term vaak tegen bij veel banen en mensen die werkzaam zijn op het gebied van digitale media, maar wat houdt deze term tegenwoordig nog in? Het web is inmiddels heel groot gegroeid, en blijft zichzelf uitbreiden met veel nieuwe updates, packages en allerlei andere dingen waarmee de "Front-End dev" te maken heeft. Is één term voor dit grote vakgebied nog wel juist?

De vele kanten in het web

Het web van tegenwoordig heeft vandaag veel verschillende kanten. Zo zijn er Back-End devs, Front-End devs, vormgevers, ux designers, bloggers, social media managers en ga zo maar door. Front-End development springt me hierbij het meest uit. Dit komt omdat het een breed gebied is waarin veel verschilende onderwerpen naar voren kunnen komen. Als Front-End developer op vandaag de dag is het bijna onmogelijk om het hele vakgebied uitstekend te beheersen. Grofweg valt het vakgebied op te delen in 2 richtingen, de JavaScript Wizards en CSS Magicians.

Source

Javascript Wizards

Een van de "richtingen" binnen Front-End development is de JavaScript kant. Aan deze kant van het vakgebied wordt er vaak gebruik gemaakt van frameworks, plugins en web applications. De Wizards focussen zich vooral op het in elkaar zetten van schone bruikbare JavaScript code, en praten graag over grote applications die ze in het verleden gemaakt hebben. De aandacht gaat aan deze kant vooral veel naar het programmeren en maken van een applicatie, en vaak minder naar het semantisch opmaken en designen van een website. CSS kennis is aanwezig tot op zekere maten, maar vaak wordt er eerst gekeken naar een JavaScript oplossing voor dingen die ook met CSS zouden kunnen.

Source

CSS Magicians

Dit is de andere kant van het Front-End gebied. CSS Magicians zijn voornamelijk, de naam zegt het al, bezig op CSS gebied. De Wizards weten elk kleine detail te fixen door middel van HTML & CSS, en houden daarbij iedere gebruiker in hun achterhoofd. In tegenstelling tot de Wizards hebben de Magicians het vaak over crazy layouts en mooie pure CSS oplossingen. Ook JavaScript gaat ze prima af, al is dit vaak alleen om de interactie op een pagina te verbeteren. De Magicians kunnen dus wel een aantal lijntjes JavaScript schrijven, maar komen liever niet al te vaak bij de web application kant van deze programmeertaal.

Maar wat is dan "de Front-end eveloper"?

Ik zie de term "Front-End developer" zelf meer als een verzamelnaam. Als je even snel wilt uitleggen waar je werkzaam bent komt te term Front-End development goed op zijn plaats. Mensen met minder kennis uit dit vakgebied snappen vaak met die term snel in wat voor sector je bezig bent en wat je ongeveer doet. Wanneer je deze term opnoemt tegen een andere Front-End developer komt er vaak toch al een gesprek aan waar je beter uitlegt aan welke kant van het spectrum jij staat.
